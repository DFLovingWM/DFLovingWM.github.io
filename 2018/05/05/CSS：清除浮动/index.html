清除浮动的几种思路

Posted on 2018-05-05

设置了浮动（float）的子元素会脱离文档流，从而使父元素塌陷，从而影响整体布局。所以，很多时候我们需要清除浮动。举个例子，一个简单的页面：

container因其子元素浮动而塌陷，所以height将会是0。

而清除浮动，通常有下面几种思路。

1. 使用clear属性

在浮动元素之后，添加带有该属性的同级block元素（例如空div），该block元素因为强制换行导致父容器被撑开。

该方法的好处就是不用像上面那样，添加一个空元素，影响DOM树结构的可读性。不过需要注意的是，除了要写clear，还要声明display、content等属性。

2. 触发“块格式化上下文”（BFC）

BFC，我理解为具有某些特性的相对独立的区域，它的其中一个特性是“闭合浮动”，即BFC高度会包含其浮动元素。因此，可以通过对父容器触发BFC，来使其包含住浮动的子元素。而触发BFC的方式有很多，比如：

  • display = inline-block | table-cell | table-caption | flex | table-flex
  • 浮动：float != none
  • 绝对定位：position = absolute | fixed
  • overflow != visible

我觉得使用overflow的影响是最小的，因为其它元素都会明显改变布局，像浮动、绝对定位又使父元素脱离文档流了，所以还是使用overflow吧：
